## Known drift / open follow-ups (do NOT block SLICE-1 done)

D1 — `parseListParams` builds the ts upper bound as `time.Now().UTC()` with strict
     `ts < ?`. clickhouse-go can round Time params such that spans written within
     ~1s of "now" are excluded from a default-window list. Surfaced during T13.
     User-visible symptom: refresh the UI within the first second after ingest
     and the list is empty.

D2 — `_schema_migrations` re-running after replay is now safe (T1 review fix added
     `CREATE TABLE IF NOT EXISTS` + `CREATE ROW POLICY IF NOT EXISTS`). Future CH
     migrations must also use `IF NOT EXISTS` on every DDL — enforce in spec review.

D3 — `cmd/seed-traces` honors `INGESTER_OTLP_GRPC_HOST_PORT` for local SignOz
     collision but does not auto-discover. CI uses default :4317. If someone runs
     `make seed-traces` against a non-default-port stack without the env var, it
     silently 401s. Low impact; an explicit `--target` flag is the workaround.

D4 — frontend/nginx.conf only proxied `/api/*` to gateway. Caddy split was
     mirrored only at the 443 ingress, so the SPA loaded from the frontend host
     port (13000 locally) could not reach query:8081 — `/api/v1/traces*` returned
     404. Caught by T16 Playwright run. Fix: added a `location /api/v1/traces`
     block in nginx.conf with `proxy_pass http://query:8081/v1/traces;` that wins
     over the generic `/api/` location. Future query-side routes (logs, services,
     topology) will need the same dual-write in both Caddyfile and nginx.conf —
     candidate for SLICE-2 consolidation (single config source or eliminate the
     nginx hop entirely).
