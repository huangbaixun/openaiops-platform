    Fixed in 30b2ce6.

    Implementer caught during cross-tenant test compile + fixed.
5. T9 — Pre-existing make lint-ch failure on cmd/topo-engine/main.go
    (Rule 2 too narrow — forbade chquery.AdminConn construction in any cmd/).
    Fixed by extending the rule to allow cmd/topo-engine/ subsystem (commit d005814).
6. T10 — d3-force forceLink needs explicit source/target fields; GraphEdge
    clone projection added in useForceSimulation composable.
7. T11 — vue-router test harness needs `await router.isReady()` before mount;
    `@/` alias not configured so tests use relative imports.
8. T13 — useForceSimulation TS2344 from T10 baseline. GraphNode + GraphEdge now
    extend d3-force SimulationNodeDatum / SimulationLinkDatum via the SimEdge
    intermediate type. Commit abff6f4. npm run build clean.
9. T15 (NEW THIS COMMIT) — services.ts + topology.ts originally landed in T11 using
    raw fetch() instead of the shared api/client.ts axios instance. The shared
    client's request interceptor (auto-injects Bearer from localStorage.apiKey) was
    therefore skipped — every /overview / /topology / /services/:name page hit 401.
    Caught only when SLICE-3 T15 Playwright run rendered the "services 401" NAlert
    error. Fixed by switching both API modules to client.get(); regression locked
    by three new vitest specs:
      api/__tests__/client.spec.ts — asserts the interceptor injects Bearer from
        localStorage (and omits the header when no key set, and baseURL=/api).
      api/__tests__/services.spec.ts — asserts client.get('/v1/services', …)
        (replacing the previous fetch('/api/v1/services?…') assertion).
      api/__tests__/topology.spec.ts — same pattern for /v1/topology.
    This bug also surfaced 3 stale stack containers (query + topo-engine + frontend)
    needing rebuild after T9/T11 — addressed by `docker compose build` + `up -d` for
    those services as part of the T15 verification loop. No production impact (the
    bug was always blocked by 401 on the first call).
