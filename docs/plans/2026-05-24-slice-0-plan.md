# Platform Rewrite · Slice 0 (Foundation) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Source spec:** `docs/specs/2026-05-24-platform-rewrite-design.md` (in archived OpenAIOps repo)
**ADR:** `docs/decisions/0007-archive-and-rewrite-as-platform.md`
**Target repo:** **https://github.com/huangbaixun/openaiops-platform** ← all paths in this plan are relative to this NEW repo, not the archived OpenAIOps repo

**Goal:** Stand up the empty platform's skeleton end-to-end: a docker-compose stack (CH + PG + Redis + OTel Collector + Caddy) plus a minimal Go gateway with Bearer auth + tenant resolution, and a Vue/TS frontend shell with Login page and i18n. Acceptance is a single curl: `curl -H "Authorization: Bearer <test-key>" http://localhost:8080/healthz` returns `{"status":"ok","tenant_id":"<uuid>"}`.

**Architecture:** Mono-repo `backend/` (Go 1.22 + chi v5 + pgx v5 + goose v3) + `frontend/` (Vue 3.4 + Vite 5 + Pinia + Naive UI + vue-i18n 9) + `deploy/` (docker-compose with ClickHouse 23.x / Postgres 16 / Redis 7 / OTel Collector 0.96 / Caddy 2.7). Multi-tenant from day 1: all PG metadata tables carry `tenant_id`, gateway resolves API key → tenant_id and injects into request context for all downstream services.

**Tech Stack:**
- Backend: Go 1.22, chi v5, pgx v5, goose v3, bcrypt, testify, slog
- Frontend: Vue 3.4, Vite 5, TypeScript 5.4, Pinia 2, Vue Router 4, Naive UI 2.38, vue-i18n 9, vitest 1.4, Playwright 1.42
- Infra: docker-compose v2, ClickHouse 23.x, Postgres 16, Redis 7, OTel Collector contrib 0.96, Caddy 2.7
- CI: GitHub Actions

## Acceptance Criteria (verbatim from spec §5 Slice 0 + expansion)

Note: `features.json` is fully superseded by ADR-0007. AC source for this plan is the spec itself.

1. **AC-1** `docker-compose up -d` 起栈，所有 service 进入 healthy 状态（CH, PG, Redis, OTel Collector, Caddy, gateway, frontend）
2. **AC-2** `curl -H "Authorization: Bearer <test-key>" http://localhost:8080/healthz` 返回 `{"status":"ok","tenant_id":"<resolved-uuid>"}`，HTTP 200
3. **AC-3** 同上但无 Authorization header 或 key 不在 PG → HTTP 401
4. **AC-4** 浏览器访问 https://localhost (Caddy TLS, 自签证书可接受) → 看到 Login 页（中文，可右上角切英文）
5. **AC-5** Login 页输入 test-key → 路由跳转到 Home 占位页 + 顶栏显示当前租户名
6. **AC-6** GitHub Actions 三个 job（`go test ./...` / `npm run test` (vitest) / `npx playwright test`）在 push 时全绿
7. **AC-7** `goose -dir backend/migrations postgres "<dsn>" up` 创建 3 张表；`goose ... down` 回滚成功
8. **AC-8** 多租户基础校验：种子两个租户的 API key，互相用对方 key 调 `/healthz` 各自返回各自 tenant_id（不串）

```go
\t\tSELECT k.id, k.tenant_id, k.name, k.hashed_key, k.scope, k.revoked_at, k.last_used_at, k.created_at,
\t\t       t.id, t.name, t.plan, t.rate_limit_per_min, t.data_retention_days, t.created_at
```

- [ ] **Step 7: Implement `stores/auth.ts`**

```ts
import { defineStore } from 'pinia'
import client from '../api/client'

interface State {
  tenantId: string | null
  tenantName: string | null
}

export const useAuthStore = defineStore('auth', {
  state: (): State => ({
    tenantId: null,
    tenantName: null,
  }),
  getters: {
    isAuthenticated: (s) => s.tenantId !== null,
  },
  actions: {
    async login(apiKey: string) {
      localStorage.setItem('apiKey', apiKey)
      try {
        const { data } = await client.get('/healthz')
        this.tenantId = data.tenant_id
        this.tenantName = data.tenant_name
      } catch (e) {
        localStorage.removeItem('apiKey')
        this.tenantId = null
        this.tenantName = null
        throw e
      }
    },
    logout() {
      localStorage.removeItem('apiKey')
      this.tenantId = null
      this.tenantName = null
    },
    async restore() {
      const key = localStorage.getItem('apiKey')
      if (!key) return
      try {
        const { data } = await client.get('/healthz')
        this.tenantId = data.tenant_id
        this.tenantName = data.tenant_name
      } catch {
        this.logout()
      }
    },
  },
})
```

```ts
        { path: '', name: 'home', component: () => import('../views/HomeView.vue') },
      ],
    },
  ],
})

router.beforeEach(async (to) => {
  const auth = useAuthStore()
  if (!auth.isAuthenticated) {
    await auth.restore()
  }
  if (to.meta.requiresAuth && !auth.isAuthenticated) {
    return { name: 'login' }
  }
  if (to.name === 'login' && auth.isAuthenticated) {
    return { name: 'home' }
  }
})

export default router
```
